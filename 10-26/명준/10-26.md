# <10.26(토) 내용 정리>

## Commit
### 로컬 저장소에서 변경사항을 기록하는 과정

## Push
### 로컬 저장소의 변경사항을 원격 저장소(GitHub)에 업로드하는 과정

## Pull
### 원격 저장소의 변경사항을 로컬 저장소로 가져오는 과정

## Branch
### Main Branch에 영향을 주지않고 새로운 기능이나 버그 등을 안전하게 수정할 수 있음

## Hotfix 
### 제품 사용 중에 발생하는 버그의 수정이나 취약점 보완, 또는 성능 향상을 위해 긴급히 배포되는 패치 프로그램

## Release 
### 개발 서버에서 해당 기능이 문제없음을 확인 후 운영서버에 적용시키는 것 (배포)
